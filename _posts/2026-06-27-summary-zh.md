## [Headroom：将 LLM 输入压缩 60-95%且不损失回答质量](https://github.com/headroomlabs-ai/headroom) ⭐️ 7.0/10

GitHub 上发布了一款名为 Headroom 的新 Python 工具，它能在工具输出、日志、文件和 RAG 文本块到达 LLM 之前对其进行压缩，将 token 使用量减少 60-95%，同时保持回答质量。它可以作为库、代理或 MCP 服务器使用。 这解决了 LLM 应用中 token 消耗高这一关键且成本高昂的问题，能直接降低 API 费用并缓解上下文窗口限制。它为构建 RAG 流程或智能体系统的开发者提供了一个实用的、即插即用的解决方案。 Headroom 是一个新仓库，社区验证有限（17 颗星，无复刻），且没有发布性能基准测试或详细的技术方法论。它通过多种方式集成，包括模型上下文协议（MCP），这使其能够作为 AI 助手的标准化服务器运行。

ossinsight · headroomlabs-ai · 6月27日 04:06

**背景**: 大型语言模型（LLM）以称为 token 的单元处理文本，供应商根据处理的 token 数量收费。检索增强生成（RAG）通过从外部文档中检索相关信息来增强 LLM，这些文档首先被分解成称为“块”的更小片段。模型上下文协议（MCP）是一个开放标准，定义了应用程序如何以一致的方式向 LLM 提供上下文。
